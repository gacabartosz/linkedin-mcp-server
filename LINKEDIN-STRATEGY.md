# LinkedIn Content Strategy — Bartosz Gaca

## Kim jestem na LinkedIn

Bartek Gaca. Builder. Buduję narzędzia AI i automatyzację. Nie jestem influencerem, nie jestem guru.
Pokazuję co buduję, jak to działa, i co nie zadziałało.

## 2 nisze (TYLKO te)

### 1. AI-powered automation tools
- MCP (Model Context Protocol) — serwery, narzędzia, integracje
- Claude Code jako narzędzie developerskie
- Automatyzacja procesów (LinkedIn, SEO, faktury)
- Open source: linkedin-mcp-server, seo-gaca-mcp, ksef-mcp, gaca-core

### 2. Polish e-gov / open data
- KSeF (Krajowy System e-Faktur) — automatyzacja, problemy, bugi
- Rejestr Cen Nieruchomości — SprawdzNotariusza.pl
- Dane publiczne + AI = transparentność
- Cyfryzacja urzędów, absurdy biurokracji

Każdy post MUSI pasować do jednej z tych nisz. Jeśli nie pasuje — nie publikuj.

## Jak piszę (tone of voice)

- **Bezpośrednio** — żadnego "I'm thrilled to announce"
- **Konkretnie** — liczby, daty, nazwy. Nie "dużo" tylko "10 341"
- **Z ironią** — "Kuzyn poprosił o jedną rzecz. Claude Code dał mu trochę więcej."
- **Po polsku** — 80% postów po polsku. Moja sieć jest polska.
- **Jak developer** — nie jak marketingowiec. Krótko, technicznie, ale zrozumiale.

## Zakazane wzorce

| NIE pisz | Dlaczego |
|----------|----------|
| "I built X in Y hours" | Brzmi jak każdy AI influencer na LinkedIn |
| "X lessons/rules/tips" | Engagement bait, algorytm penalizuje |
| "Let me tell you a story" | Sztuczne, manipulacyjne |
| "Game changer / disrupting" | Buzzwordy bez treści |
| Generyczne bannery z Puppeteer | Wyglądają jak reklamy, -47% reach |
| Link w treści posta | -40% reach. Link TYLKO w komentarzu |
| 5+ hashtagów | -68% reach |
| 2 posty tego samego dnia | Algorytm penalizuje >1/12h |

## Struktura posta

```
[HOOK — max 62 znaki, problem lub porażka]

[KONTEKST — co się stało, 2-3 zdania]

[ROZWIĄZANIE — co zrobiłeś, konkrety]

[INSIGHT — co z tego wynika, czego się nauczyłeś]

[PYTANIE — jedno, prawdziwe, nie bait]

#hashtag1 #hashtag2 #hashtag3
```

### Hook — najważniejsze 62 znaki

To co widać na mobile PRZED "see more". Decyduje o 90% kliknięć.

**Dobre hooki:**
- "Ministerstwo Sprawiedliwości udostępnia dane notariuszy." (fakt + ciekawość)
- "Mój kuzyn poprosił o jedną rzecz." (historia + napięcie)
- "Faktury korygujące w EUR to pułapka." (problem + specyfika)
- "Zautomatyzowałem LinkedIn. Efekt mnie zaskoczył." (outcome + zaskoczenie)

**Złe hooki:**
- "I built 29 tools in 48 hours" (przechwałka)
- "5 lessons from building MCP tools" (listicle bait)
- "Here's what I learned" (generic)

## Format mediów (priorytet)

1. **Screenshot z terminala/kodu** — autentyczny, developer-friendly
2. **Screenshot z działającej aplikacji** — dowód że działa
3. **PDF carousel z oryginalnymi danymi** — case study, framework
4. **Tekst bez mediów** — storytelling, refleksja, porażka
5. **Video z nagrania ekranu** — demo, walkthrough

**NIGDY:** generowane bannery, stock photos, AI-generated images, Canva templates

## Komentarze (auto-comment strategy)

Auto-komentarz powinien:
- Dodawać wartość (ciekawostka, kontekst techniczny, dane)
- NIE zaczynać się od linku
- Link na końcu, naturalnie, jeśli pasuje
- Min 20 słów
- Nie każdy post potrzebuje komentarza z linkiem

**Dobry komentarz:**
"Cały ranking opiera się na danych z Rejestru Cen Nieruchomości — publicznej bazy Ministerstwa Sprawiedliwości. Każda transakcja notarialna od 2013 roku. Jeśli chcesz sprawdzić mediany cen w swoim mieście: sprawdznotariusza.pl"

**Zły komentarz:**
"SprawdzNotariusza.pl to open-source side project. Stack: Node.js + PostgreSQL + Leaflet. https://sprawdznotariusza.pl https://github.com/gacabartosz/..."

## @Mentions

Używaj prawdziwych @mentions (ugcPosts attributes API), NIE "Nazwa (link)".

Tagujesz TYLKO gdy:
- Post dotyczy tej organizacji bezpośrednio
- Chcesz ich uwagę (strategicznie, nie spamersko)
- Max 2-3 mentions na post

## Kalendarz publikacji

- **Wtorek, środa, czwartek** — najlepsze dni
- **07:30 lub 08:00 CET** — optimum dla polskiej sieci
- **3 posty tygodniowo** max
- **Min 18h między postami**
- **Golden hour** — 60 min po publikacji być dostępnym na komentarze

## Metryki sukcesu

| Metryka | Target | Dlaczego |
